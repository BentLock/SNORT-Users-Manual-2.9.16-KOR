### 3.2.1 Action

Action 필드는 헤더와 옵션의 모든 속성을 가진 패킷이 나타날 경우 어떻게 해야 하는지를 정의합니다. Rules의 첫 번째 항목입니다. 기본적으로 alert, log, pass가 있으며, 인라인 모드로 Snort가 동작한다면 *(\*예를 들어 IPS에서)* \*drop, \*reject, \*sdrop의 동작들이 있습니다.

1. pass - 패킷을 무시하고 통과시킨다.

2. \* drop - 패킷을 차단하고 기록한다.

3. \* reject - 패킷을 차단하고 로그를 기록한 후, 프로토콜에 따른 접근 불가 패킷을 보낸다. (TCP : tcp reset, UDP/ICMP : ICMP-error)

4. alert - 선택한 알림 메서드를 사용하여 alert를 생성하고, 패킷을 기록한다.

5. log - 패킷을 로그에 저장한다.

6. \* sdrop(Silent Drop) - drop과 동일한 동작을 수행하지만, 기록하지 않는다.


*\* 액션에는 기본적인 우선순위가 있습니다. pass - drop - reject - alert - log 순서로 우선순위가 높아 이를 이용해 화이트리스트 방식으로 룰셋을 관리할 수 있습니다.*

또한, 사용자 지정(your own) rule types를 정의하고, 하나 이상의 출력 플러그인과 연결시킬 수 있습니다. 이 사용자 지정정 rule types를 룰에서 action으로 사용할 수 있습니다.

#### example)
아래 예시는 syslog와 tcpdump에 기록하는 rule types를 정의합니다.
```yaml
ruletype redalert
{
    type alert
    output alert_syslog: LOG_AUTH LOG_ALERT
    output log_tcpdump: suspicious.log
}
```

### 3.2.2 Protocols

다음 필드는 프로토콜입니다. Snort 2.9.16 에서는 네 가지 프로토콜을 설정할 수 있습니다. - TCP, UDP, ICMP, IP

### 3.2.3 IP Addresses

룰의 IP 주소를 다룹니다. 키워드 `any`는 모든 주소를 정의합니다.

단일 IP 주소, 여러 IP 주소, CIDR 블록으로 구성할 수 있습니다. *(snort2 에서는 IP 리스트 내 공백을 허용하지 않습니다. ex: [192.168.1.1, 192.168.1.10] -> fatal error)*

또한, 부정 연산자`!`가 있습니다.

#### example 1)
이 룰은 IP 주소가 내부 네트워크에서 출발하지 않고, 목적지 주소가 내부 네트워크의 111포트인 tcp 패킷을 나타냅니다.
```snort
alert tcp !192.168.1.0/24 any -> 192.168.1.0/24 111 \
    (msg:"external mountd access"; content:"|00 01 86 a5|";)
```

#### example 2)
이 룰룰은 IP 주소 목록을 지정했습니다. IP 리스트는 쉼표(`,`)로 구분된 IP 주소 목록과 CIDR 블록을 대괄호로 감싸서 지정합니다. 
```snort
alert tcp ![192.168.1.0/24,10.1.1.0/24] any -> \
    [192.168.1.0/24,10.1.1.0/24] 111 (msg:"external mountd access"; \
    content:"|00 01 86 a5|";)
```

### 3.2.4 Port Numbers

포트 번호는 여러 방식으로 지정할 수 있습니다. `any` 포트, 정적 포트 정의 - 단일 혹은 멀티(`[ ]`), 범위 지정(`:`), 부정(`!`) 등이 포함됩니다.

#### example 1)

이 룰은 UDP 패킷의 모든 출발지 ip, port에서 192.168.1.0/24의 1~1024 포트로 향하는 패킷을 지정합니다.
```snort
alert udp any any -> 192.168.1.0/24 1:1024
```

#### example 2)

이 룰은 tcp의 모든 ip의 0~1024 포트에서 출발하고, 목적지가 192.168.1.0/24의 80 포트인 패킷을 지정합니다.
```snort
alert tcp any :1024 -> 192.168.1.0/24 80
```

#### example 3)

이 룰은 tcp의 192.168.1.0/24 범위가 아닌 IP 주소의 500번 이상 포트에서 출발하고, 목적지가 192.168.1.0/24 주소의 80와 8080포트를 제외한 포트인 패킷을 지정합니다.
```snort
alert tcp ![192.168.1.0/24] 500: -> 192.168.1.0/24 ![80, 8080]
```

### 3.2.5 The Direction Operator

방향 연산자는 단방향(`->`)과 양방향(`<>`)이 있습니다. 일반적으로 방향 연산자 앞에 출발지 정보가 있고, 뒤에 목적지 정보가 있습니다.

1.8.7 이전 Snort 버전에서는 방향 연산자가 적절한 오류 검사를 하지 않았고, 룰이 항상 일관되게 읽히게 하기 위해 `<-` 연산자가 **존재하지 않습니다.**
